<p>We're taking a brief break from the regular "Interesting Ruby Tidbits" posts to focus on Rails™ only for a change. Rails has just made it to version 2.0 so there's been plenty of action in the community. Here are some of the highlights:</p>
<p><strong>The First Rails 2.0 Screencast</strong></p>
<p>I can't confirm the validity of its title, but <a href="http://www.akitaonrails.com/2007/12/10/the-first-rails-2-0-screencast-english">The First Rails 2.0 Screencast</a> is a screencast by Fabio Akita that walks through building a RESTful blog system in 30 minutes using Rails 2.0.</p>
<p><strong>PeepCode Rails2 PDF (now in Spanish too!)</strong></p>
<p><strong><img src="/assets/2007/12/rails2bookes.png" width="216" height="114"><br></strong></p>
<p>"<a href="http://peepcode.com/products/rails2-pdf">Rails2</a>" is a 58-page e-book (available in PDF format) by Ryan Daigle and published by Geoffrey Grosenbach's Peepcode that demonstrates how to use Rails 2.0's new features. A <a href="https://peepcode.com/products/rails-2-pdf-es">Spanish version</a> has also now been published. It's available por solo $9 por senors y senoritas.</p>
<p><strong>BackgroundFu - Background task runner plugin for Rails that isn't BackgrounDRB</strong></p>
<p><a href="http://trix.pl/blog/ruby-on-rails/running-long-background-tasks-in-ruby-on-rails-made-dead-simple">BackgroundFu</a> is a new Rails plugin that enables you to run arbitrary background / long-running tasks in relation to a Rails application. It differs from the more established <a href="http://backgroundrb.rubyforge.org/">BackgrounDRB</a> by being more suitable for arbitrary, on-the-go long running tasks, rather than routine background tasks. There appear to be a number of downsides to BackgroundFu's approach, however, which are clarified in the comments associated with <a href="http://trix.pl/blog/ruby-on-rails/running-long-background-tasks-in-ruby-on-rails-made-dead-simple">this post</a>. One worth watching for the future though.</p>
<p><strong>10 Ruby on Rails Plugins You Should Be Using</strong></p>
<p><a href="http://hosting.media72.co.uk/blog/2007/12/09/10-ruby-on-rails-plugins-you-should-be-using/">10 Ruby on Rails Plugins You Should Be Using</a> is an article that pretty much covers what it says in the title. The author looks at ten different useful Rails plugins and gives a reason for their inclusion in the list. Amongst the ten are attachment_fu, white_list, Rspec and autotest.</p>
<p><strong>Rendering with Erubis on Rails 2.0</strong></p>
<p>In <a href="http://railspikes.com/2007/12/10/rendering-erubis-and-rails-2-0">Rendering with Erubis and Rails 2.0</a>, Eric X shows how to replace the default Erb rendering library with the faster, more powerful <a href="http://www.kuwata-lab.com/erubis/">Erubis</a>.</p>
<p><strong>Declarative Exception-Handling in Rails 2.0</strong></p>
<p>Chu Yeow <a href="http://blog.codefront.net/2007/12/10/declarative-exception-handling-in-your-controllers-rails-20-a-feature-a-day-2/">demonstrates</a> the very clean way in which you can handle controller-level exceptions in Rails 2.0.</p>
