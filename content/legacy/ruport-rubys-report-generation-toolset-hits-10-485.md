<p><img src="/assets/2007/05/ruport.jpg" height="82" width="258" border="1" hspace="4" vspace="4" alt="Ruport"></p>
<p>Gregory Brown has just <a href="http://www.oreillynet.com/ruby/blog/2007/05/post_1.html">announced</a> the release of <a href="http://rubyreports.org/">Ruport</a> 1.0, a collection of tools that provide "reporting" functions within Ruby. A Rails plugin, <a href="http://stonecode.svnrepository.com/ruport/trac.cgi/wiki/ActsAsReportable">acts_as_reportable</a>, is also available to make report generation from Rails applications easy. Reporting tools work by taking your data, processing it using a report definition, then provide output in various formats (on-screen, to file - such as PDF, to print, etc).</p>
<p><img src="/assets/2007/05/ruport2.jpg" height="266" width="180" border="1" hspace="4" vspace="4" alt="Ruport2"></p>
<p>Gregory has put together <a href="http://www.oreillynet.com/ruby/blog/2007/05/post_1.html">a short example and release notes</a> for anyone who's not familiar with the concept over at the O'Reilly Ruby blog, and there are <a href="http://reporting.stonecode.org/ruport-rubyconf-06.pdf">also</a> <a href="http://rubyreports.org/presentations/boston-2007.02.13.pdf">some</a> <a href="http://concentrationstudios.com/assets/2007/2/28/ruport.pdf">presentations</a> for your viewing pleasure.</p>
