<p><img src="/assets/2010/12/apple-ruby-3.jpg" width="69" height="83" alt="apple-ruby-3.jpg" style="float:left; margin-right:12px; margin-bottom:12px;">MacRuby team lead and Apple employee Laurent Sansonetti has <a href="http://lists.macosforge.org/pipermail/macruby-devel/2010-December/006594.html">unveiled MacRuby 0.8</a>, the latest release of the Ruby 1.9-compatible Mac-focused Ruby implementation. The <a href="http://www.macruby.org/">main MacRuby site</a> hasn't been updated at the time of writing but you can get MacRuby 0.8 <i>now</i> from the <a href="http://www.macruby.org/files/">MacRuby files folder</a> or direct at <code><a href="http://www.macruby.org/files/MacRuby%200.8.zip">http://www.macruby.org/files/MacRuby 0.8.zip</a></code></p>
<p>0.8 follows on swiftly from the <a href="http://lists.macosforge.org/pipermail/macruby-devel/2010-December/006550.html">MacRuby Bug Smash</a> held on IRC on Saturday 4th December. Given this, <b>0.8 is primarily a bug fixing release</b> with more compatibility and stability fixes than you could shake a bacon-coated stick at. Some enhancements have been made to MacRuby's Cocoa development support (a post on the <a href="http://www.macruby.org/blog/index.html">official MacRuby blog</a> is due soon) and <i>macirb</i> now supports auto-indentation and completion.</p>
<p><em><strong>Update:</strong> Wayne E Seguin has released an updated version of RVM - 1.1.5 - that uses MacRuby 0.8 by default for its MacRuby implementation installer.</em></p>
<p>As well as a new MacRuby release, there's also some news regarding a couple of new MacRuby books. They're a perfect way to get going with this now production ready implementation (and it is - there are a lot of OS X GUI app projects underway now, such as Erik Michaels-Ober's <a href="http://www.kickstarter.com/projects/1487030260/hubcap-a-github-client-for-mac-os-x">Hubcap</a>).</p>
<h3>MacRuby: The Definitive Guide</h3>
<p><img src="/assets/2010/12/rc_cat.gif" width="91" height="119" alt="rc_cat.gif" style="float:right; margin-bottom:12px; margin-left:12px;"> First up - and the most complete - is <a href="http://oreilly.com/catalog/0636920000723/"><b>MacRuby: The Definitive Guide</b></a> by Matt Aimonetti (who's been quite involved with the MacRuby project for a while now). It's technically still in beta but the "early release e-book" edition is now available to buy and the print version (due in April 2011) can be preordered.</p>
<p>Matt's book is interesting because it's also <a href="http://ofps.oreilly.com/titles/9781449380373/">available in a free, Creative Commons licensed form</a> through O'Reilly's Open Feedback Publishing System. Nonetheless, you're encouraged to buy the official release if you are able.</p>
<h3>MacRuby in Action</h3>
<p><img src="/assets/2010/12/lim_cover150.jpg" width="88" height="109" alt="lim_cover150.jpg" style="float:right; margin-bottom:12px; margin-left:12px;"></p>
<p><b><a href="http://www.manning.com/lim/">MacRuby in Action</a></b> is another MacRuby book - this time by <i>Brendan Lim</i> (of <a href="http://intridea.com/">Intridea</a> fame) and <i>Paul Crawford</i>. Very little of the book has been released under Manning's Early Access Program so far, alas, and the final print date is "Summer 2011." Anyone who owns a few Manning books know just how well they're put together though, so MacRuby in Action shouldn't disappoint.</p>
<p>I also have word from Brendan Lim that if you use the code <code>macruby50</code> at checkout <i>before January 4, 2011</i> <b>you'll get a whopping 50% off</b> - so if you're going to buy this book anyway, you might as well do it while this offer is valid.</p>
<p class="s" style="padding: 8px; background-color: #ff9"><em>[suggestion]</em> Want to get on the Rails 3 train? Michael Hartl's <a href="http://zfer.us/EKm97">Ruby on Rails 3 Tutorial series</a> is the way to go. There's a free, online book but if you want to go further, pick up the <a href="http://zfer.us/EKm97">15+ hours of screencasts</a> giving you an 'over the shoulder' view of a Rails professional building Rails 3 apps in real time.</p>
