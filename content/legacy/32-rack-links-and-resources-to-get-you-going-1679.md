<p><img src="/assets/2009/04/rack.png" width="82" height="96" alt="rack.png" style="float:left; margin-right:12px; margin-bottom:12px;"> If you've developed a Web application using Ruby lately, you've <em>probably</em> used <a href="http://rack.rubyforge.org/">Rack</a> in one way or another. Rack calls itself a "Ruby Web server interface" and I tend to think of it as an abstraction between the messy world of HTTP and the potentially just as messy world of your code.</p>
<p>Rack became a key element of Rails as of Rails 2.3, and has been of importance in deploying applications built on alternate frameworks (such as <a href="http://www.sinatrarb.com/">Sinatra</a>). Put it this way, Rack has become a very big deal.</p>
<p>Taking perhaps the tiniest of cues from our articles like <a href="44-merb-resources-1318.html">Merb 1.0's 44 Links and Resource to Get You Going</a> and <a href="rails-22-released-27-links-and-resources-to-get-you-going-1354.html">Rails 2.2's 27 Links and Resources To Get You Going</a>, Rails Envy star Jason Siefer has put together <strong>the splendid..</strong> <a href="http://jasonseifer.com/2009/04/08/32-rack-resources-to-get-you-started"><strong>32 Rack Resources To Get You Started</strong></a><strong>.</strong> A fine list it is too with links to top blog posts, screencasts, presentations, middleware lists, tutorials, and basic resources.</p>
<p>Anyway, <a href="http://jasonseifer.com/2009/04/08/32-rack-resources-to-get-you-started">check out Jason's list.</a> We're not going to repeat it here, but I think it carries on a fine tradition of the ever-popular resource list :)</p>
