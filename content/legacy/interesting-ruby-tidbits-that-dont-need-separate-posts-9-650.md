<p><strong>Interview with Evan Phoenix about Rubinius</strong></p>
<p>Mike Moore, of the <a href="http://rubiverse.com/">Rubiverse Podcast</a>, has <a href="http://rubiverse.com/podcasts/3-evan-phoenix-on-rubinius">interviewed legendary Ruby developer Evan Phoenix</a> about the <a href="http://rubini.us/">Rubinius</a> Ruby implementation project. If you like your Ruby talk deep down, dirty, and packed full of technical details, this is well worth a listen. The interview goes right into the inspiration behind Rubinius, the development process, and stories of discoveries made so far.</p>
<p><strong>Make A Code Pasting Site In 15 Minutes With Sinatra &amp; DataMapper</strong></p>
<p>Nick Plante has written <a href="http://blog.zerosum.org/2007/11/5/build-a-simple-pastie-clone-with-sinatra-datamapper-in-15-minutes">a great tutorial demonstrating how to build a code pasting site</a> (like <a href="http://pastie.caboo.se/">Pastie</a>) in a very short period of time using <a href="http://sinatra.rubyforge.org/">Sinatra</a>, a new lightweight Ruby Web framework, and <a href="http://datamapper.org/">DataMapper</a>, a compelling new ActiveRecord alternative (which is going to be featured a lot more on Ruby Inside shortly).</p>
<p><strong>E-mail Validation Rails Plugins</strong></p>
<p>Carsten Nielsen has developed an "<a href="http://rails.savvica.com/2007/11/6/email-veracity-plugin">Email Veracity Plugin</a>" for Rails that makes it really easy to validate e-mail addresses supplied within Rails applications. It can even check the MX records associated with the e-mail address supplied to see if the domain exists and is accepting mail! Basically, if you care about e-mail address validity, it's a must-install.</p>
<p><strong>make_resourceful 0.2.0 Released</strong></p>
<p>Hampton Catlin has <a href="http://nex-3.com/posts/54-make_resourceful-0-2-0">released version 0.2.0 of make_resourceful</a>, an incredibly useful Rails plugin that makes it a snap to set up a whole bunch of RESTful actions and responses in a controller with merely one line of code. A new documentation site for the plugin is also now available at <a href="http://mr.hamptoncatlin.com/">http://mr.hamptoncatlin.com/</a></p>
<p><strong>Testing XML Builder Views in Rails</strong></p>
<p>Matt Aimonetti looks at <a href="http://railsontherun.com/2007/10/31/how-to-test-a-xml-builder-view">how to test XML Builder views within Rails applications</a> using RSpec and Hpricot. This is not an area I'm particularly familiar with, but supposedly it's not particularly easy to figure out from scratch, but Matt's resulting code looks sharp.</p>
