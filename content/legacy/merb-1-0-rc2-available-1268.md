<p><img src="/assets/2008/10/merblogo.png" width="137" height="55" alt="merblogo.png" style="float:left; margin-right:12px; margin-bottom:12px;"> Following on from the highly successful MerbCamp, the <a href="http://merbivore.com/">Merb Framework</a> took another big step this week with the announcement of the availability of release candidate 2 for it's upcoming 1.0 release (due within weeks now). No major changes and The <a href="http://merbist.com/2008/10/21/merb-10-rc2/">Merbist blog</a> lists mainly a focus on bug fixes and Windows compatibility:</p>
<blockquote>
<p>The main focus for this release was to fix bugs and make the stack Windows compatible. We didn’t get any major bugs in RC1 but fixed a lot of small annoyances and problems with generated resources. We also made sure Merb itself would work properly with Windows (not using incompatible signals etc..) and we spent some time getting the Data Object sqlite3 drivers compiled on Windows.</p>
</blockquote>
<p>Seems like a great time to begin playing around with Merb if you haven't yet. You can install / update Merb through RubyGems using <em>sudo gem install merb</em></p>
<p>Ruby Inside is planning to run a resource packed feature on Merb when it hits 1.0, so keep your eyes peeled!</p>
