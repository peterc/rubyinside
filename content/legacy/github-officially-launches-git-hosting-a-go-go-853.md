<p>It's been the programming world's worst kept secret, but the covers are finally off as <a href="http://github.com/">GitHub</a> officially launched today. No more beta invites needed - hurrah!</p>
<p>GitHub is, officially, a <a href="http://git.or.cz/">Git</a> repository hosting service (where Git is a source code control system - think decentralized, distributed <a href="http://subversion.tigris.org/">SVN</a>) built by Chris Wanstrath (<a href="http://errtheblog.com/">Err the Blog</a>), Tom Preston-Werner (creator of <a href="http://chronic.rubyforge.org/">Chronic</a> and <a href="http://god.rubyforge.org/">God</a>), and PJ Hyett. GitHub's early users are calling it a "social programming network," a "FaceBook for coders," and all sorts of wonderful things. This is because Git's decentralized nature makes it easy to fork, branch and merge code, and so does GitHub, which makes GitHub an ideal platform for collectively working on software, especially open source.</p>
<p>As of Rails 2.1 (coming very soon), GitHub will be the new official home of the Rails source repository, and the Merb, RSpec, Capistrano, Prototype and Scriptaculous projects are also on board, along with hundreds of <a href="http://github.com/repositories">other Git repositories</a> created by beta users.</p>
<p>GitHub is a commercial offering (though from two open source devotees), but with an incredibly generous free level, meaning that the service is free for most reasonably sized open source projects (that is, the free level covers that sort of open use). There are, of course, <a href="http://github.com/plans">a wide range of plans available</a> for the whole gamut of users.</p>
<p>If you haven't yet given Git a go, don't fear, as a round up of Git tutorials, tips and tricks (with a Ruby slant, naturally) is coming next here on Ruby Inside. That said, I've just noticed InfoQ has <a href="http://www.infoq.com/news/2008/04/ruby-git-roundup-rails-rubyforge">already posted something similar</a>, so read that if you're feeling impatient.</p>
