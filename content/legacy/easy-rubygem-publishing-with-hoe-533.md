<p>Geoffrey Grosenbach, author of the famous <a href="http://www.peepcode.com/">PeepCode</a> screencasts, has written a tutorial about using Ryan Davis' <a href="http://seattlerb.rubyforge.org/hoe/">hoe</a> library to <a href="http://nubyonrails.com/articles/tutorial-publishing-rubygems-with-hoe">make releasing your own RubyGems a cakewalk</a>. Geoffrey says that using Hoe turns a laborious, multi-step twenty minute process into only a few Rake tasks. Hoe takes the automation of Gem publishing seriously, and once you've set it up properly you can go as far as let it post announcements (to your blog and/or Rubyforge), build your docs, create an e-mail announcement, run testing packages against the library, as well as actually releasing and uploading the gem to Rubyforge ready for public consumption. CPAN, eat your heart out!</p>
