<p><img src="/assets/2009/05/toolbox-red-256x256.png" width="93" height="93" alt="Toolbox_Red-256x256.png" style="float:left; margin-right:12px; margin-bottom:12px;"><a href="http://ruby-toolbox.com/">The Ruby Toolbox</a> gives Ruby developers a categorized overview of 100 or so different libraries ranked on how commonly used they are. It's not perfect as it only pays attention to projects hosted on Github and the ranking system is based on the number of watchers and forks they have, but it's enough to give you a basic overview of the activity within a certain area.</p>
<p>For example, in the <a href="http://ruby-toolbox.com/categories/browser_testing.html">Browser testing</a> section, <a href="http://github.com/brynary/webrat/tree/master">webrat</a> picks up the #1 spot with 750 watchers and 147 forks, while <a href="http://github.com/bret/watir/tree/master">watir</a> is down at #3 with only 42 watchers and 18 forks. Whether this is a fair assessment or not is down to you. Even if a project has its code on Github, not every project has "culturally" moved there yet, and so the number of followers and forks could be misleading.</p>
<p>As a way of finding out what libraries are at least getting widespread acceptance, The Ruby Toolbox is awesome, particularly in changeable areas like <a href="http://ruby-toolbox.com/categories/rails_tagging.html">Rails Tagging plugins</a> (acts_as_taggable_on takes #1) and <a href="http://ruby-toolbox.com/categories/testing_frameworks.html">Testing Frameworks</a> (Cucumber takes #1). That said, hopefully you're using better criteria than the number of followers and forks of a project to make key development decisions anyway.</p>
