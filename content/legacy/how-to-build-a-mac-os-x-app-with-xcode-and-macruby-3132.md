<p><img src="/assets/2010/03/rubyxcode.png" class="alignleft size-thumbnail bordered">Want to develop a Mac OS X app without getting waist deep in Objective C? <a href="http://www.macruby.org/">MacRuby</a> is the answer, and it’s now mature enough to use directly from XCode to build fully-featured Ruby-powered Mac apps. “Jean Pierre Hernandez” of Phusion presents <a href="http://blog.phusion.nl/2010/03/12/creating-our-very-first-mac-application-with-ruby-how-exciting/">a walkthrough of how to do it</a>, step by step.</p>
<p><a href="http://blog.phusion.nl/2010/03/12/creating-our-very-first-mac-application-with-ruby-how-exciting/">The tutorial</a> walks through:</p>
<ul>
<li>Creating a new project in XCode</li>
<li>Designing an interface with Interface Builder</li>
<li>Customizing form widgets</li>
<li>Writing a controller in MacRuby</li>
<li>Connecting the controller to the window</li>
</ul>
<p>Any knowledge of XCode, Objective C, and Interface Builder will enable you to get through this walkthough a lot quicker, but it’s not <em>essential.</em></p>
<h3>Installing MacRuby and XCode</h3>
<p>While it’s possible to install MacRuby by compiling yourself, it takes forever, so <em>Jean Pierre</em> recommends grabbing <a href="http://www.macruby.org/downloads.html">the latest stable binary release</a> and going from there. It’s quick to install and doesn’t require any messing around at the command line. Also, if you haven’t yet installed XCode from your OS X discs, now’s the time – you’ll need XCode to get anywhere with this walkthrough. You can also <a href="http://developer.apple.com/technologies/xcode.html">download them from Apple</a>.</p>
