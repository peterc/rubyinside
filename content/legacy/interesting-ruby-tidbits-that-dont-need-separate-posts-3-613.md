<p><strong>Tracking Down a Rails App Memory Leak</strong>Tom Copeland posts about <a href="http://tomcopeland.blogs.com/juniordeveloper/2007/09/tracking-down-a.html">how he tracked down a memory leak in a Rails application</a>. The results are interesting.<strong> </strong><strong>RubyGems Runs on Ruby 1.9-ish</strong>Eric Hodel reports that all of the tests for RubyGems <a href="http://blog.segment7.net/articles/2007/09/20/rubygems-tests-pass-on-1-9">now pass</a> on Ruby 1.9 and that a 1.9 compatible beta version will be due shortly.<strong>R2Check - Tiny app that checks your pre-Rails 2.0 apps for compatability</strong>Mislav Marohnic has built <a href="http://pastie.caboo.se/private/krcevozww61drdeza13e3a">a pretty cool Ruby script</a> that checks the source code of a Rails app for compatibility with Rails 2.0 and then points out the changes required.<strong>Advice for Ruby Beginners - Part 1</strong>Satish Talim has posted <a href="http://rubylearning.com/blog/2007/09/27/advice-for-ruby-beginners-1/">the first part of mass interview with 13 Ruby gurus</a> with questions surrounding topics important to Ruby newbies. The "gurus" (quoted because <em>I'm</em> in the list) include Jamis Buck, Jamie van Dyke, Ola Bini, and a whole range of people from a stunningly diverse array of countries.<strong>RubyForge vs CPAN</strong>Daniel Berger does <a href="http://www.oreillynet.com/ruby/blog/2007/09/rubyforge_vs_cpan_1.html">a quick comparison</a> of Perl's infamous CPAN library and code repository with Ruby's own RubyForge. I was pretty surprised at the numbers.. it seems RubyForge is not only winning on quality but for the size of the Ruby community versus the Perl community, the quantity is amazing.</p>
