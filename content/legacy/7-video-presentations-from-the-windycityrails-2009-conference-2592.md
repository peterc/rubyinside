<p><img src="/assets/2009/10/windycityrails.png" width="58" height="72" alt="windycityrails.png" style="float:left; margin-right:12px; margin-bottom:12px;"> The 1-day <a href="http://windycityrails.org/">WindyCityRails</a> conference happened again this year, this time in downtown Chicago at the Westin Hotel on September 12. Reviews were generally positive with the conference sessions balancing immediately-useful Ruby and Rails tips with related concepts like functional programming and design. All of the main track sessions were professionally recorded and are <a href="http://windycityrails.org/videos">available through the official website</a>, though I've linked to each of them individually below as well:</p>
<div style="margin-left: 1em">
<p><a href="http://windycityrails.org/videos#1"><strong>Better Ruby through Functional Programming</strong></a> <i>by Dean Wampler</i> – Dean Wampler is author of <a href="http://programming-scala.labs.oreilly.com/">Programming Scala</a>. He introduced functional programming concepts and how to implement them in Ruby.</p>
<p><a href="http://windycityrails.org/videos#2"><strong>Super-easy <span class="caps">PDF</span> Generation with Prawn and Prawnto</strong></a> <i>by John McCaffrey</i> – <span class="caps">PDF</span> generation is a complex feature and John described various types and recommended particular tools for each type.</p>
<p><a href="http://windycityrails.org/videos#3"><strong>Comics Is Hard: On Domains and Databases</strong></a> <i>by Ben Scofield</i> – Modeling certain domains with relational data stores breaks down quickly. Ben described two and briefly discussed <span class="caps">SQL</span>-less alternatives that work better.</p>
<p><a href="http://windycityrails.org/videos#4"><strong>UI Fundamentals for Programmers</strong></a> <i>by Ryan Singer</i> – Ryan advocated "designing first" when building applications and offered examples of design principles in action from his work at 37Signals.</p>
<p><a href="http://windycityrails.org/videos#5"><strong>How To Test Absolutely Anything (in Rails)</strong></a> <i>by Noel Rappin</i> – Noel is author of <a href="http://www.railsrx.com/">Rails Test Prescriptions</a>. His talk was a practical walkthrough of how to test each application tier, even view testing and JavaScript testing.</p>
<p><a href="http://windycityrails.org/videos#6"><strong>Optimizing Perceived Performance</strong></a> by David Eisinger – David gave a humorous talk about an example Rails and jQuery application he built that demonstrates responsive user interface patterns. The <a href="http://github.com/dce/dbdb">dbdb application is on github</a>.</p>
<p><a href="http://windycityrails.org/videos#8"><strong>Rails 3 Update</strong></a> <i>by Yehuda Katz</i> – Yehuda’s session was last. He discussed framework design in a general and how the Rails core team has addressed keeping Rails 3 easy to learn while adding new features. New features were discussed in brief and Yehuda mentioned that an alpha version of Rails 3 may be available in October.</p>
</div>
<p>One of the nice things about conferences is getting to meet some prominent figures in the community, and WindyCityRails was no exception! Thanks to the organizers for a solid event. For more conference coverage check <a href="http://blog.webandy.com/2009/09/25/windycityrails-2009.html">my blog post</a>.</p>
