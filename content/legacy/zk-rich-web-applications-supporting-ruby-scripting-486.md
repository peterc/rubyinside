<p><a title="ZK Ajax Framework" href="http://zkoss.org/">ZK</a> is a popular open source Ajax framework written in Java. It simplifies rich web applications by treating the browser as a simple display server, which reduces the need to have lots of application logic. It's Java, so why it's in RubyInside?</p>
<p>The interesting part here is that recent versions of ZK (<a href="http://zkoss.org/release/rn-2.3.1.dsp">version 2.3.1</a> as of this writing) support more languages for its ZUML markup, in addition to Java: JavaScript, Groovy, <a title="Ruby support in ZK" href="https://sourceforge.net/tracker/index.php?func=detail&amp;aid=1656785&amp;group_id=152762&amp;atid=785194">and <strong>Ruby</strong></a>. ZUML is an XML-based markup language to create user interfaces very similar to the desktop GUI component/event driven model.</p>
<p>Being Java, ZK's Ruby engine is powered by <a href="http://jruby.codehaus.org/">JRuby</a>, which is already quite mature. With the recent announcement of <a href="silverlight-and-ruby-on-the-net-clr-473.html">Microsoft Silverlight technology</a> (which also supports Ruby), this can only mean wider adoption of the Ruby language in many aspects.</p>
<p><em>(One particular note: both </em><a href="http://www.rubyonrails.org/"><em>Rails</em></a><em> and ZK </em><em><a href="http://wiki.script.aculo.us/scriptaculous/show/RealWorldUsage">use Script.aculo.us</a></em><em> as their visual effects JavaScript library.)</em></p>
