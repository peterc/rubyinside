<p><img src="/assets/2010/03/rf.png" class="alignleft size-thumbnail bordered"><a href="http://www.rubyflow.com">RubyFlow</a> is Ruby Inside's community driven sister site where you can post cool Ruby links you want to share (even of your own stuff). With 20–80 posts each week, there's too much to cover on Ruby Inside, but I want to provide a regular roundup of the "best of" RubyFlow. This instalment covers early March — enjoy!</p>

<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3596">An Interview Question That Prints Out Its Own Source Code (In Ruby)</a></strong> I recently decided to play around with <a href="http://www.skorks.com/2010/03/an-interview-question-that-prints-out-its-own-source-code-in-ruby/">writing a ruby quine</a> (especially since there didn't seem to be many around). <p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3586">Rit – The anti-<span class="caps">CMS</span> content scheduling system in Rails</a></strong> Need to support business users updating website content? You need a <span class="caps">CMS</span> right? Probably not. <a href="http://bit.ly/9dJYan">Introducing Rit. – The anti-<span class="caps">CMS</span> content scheduling system in Rails</a>. Rit. is a standalone web application that allows users to edit multiple editions of website content and designate when that content should show up on the site. Site content is served up to a consuming application as a web service.<br><small><em>by <a href="http://emphaticsolutions.com">briandoll</a></em></small></p>

<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3539">Vim for Rails Development</a></strong> Where to start and how to learn <a href="http://dalibornasevic.com/posts/11-vim-for-rails-development">Vim for Rails Development</a> by referencing some books/screencasts.<br><small><em>by <a href="http://dalibornasevic.com">dalibor</a></em></small></p>
<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3581">Spree 0.10.0 Released</a></strong> We've just released the newest version of <a href="http://spreecommerce.com">Spree</a> which is an open source e-commerce framework for Ruby on Rails. Highlights include: theming, <span class="caps">REST</span> <span class="caps">API</span> and Ruby 1.9 support. See the <a href="http://spreecommerce.com/blog/2010/03/13/spree-0100-released/">release announcement</a> for more details.<br><small><em>by <a href="http://railsdog.com">railsdog</a></em></small></p>
<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3580">Install any Java library as a JRuby gem</a></strong> Charles Nutter said <em>"… a prototype Maven server that looks and feels like a RubyGems source. By setting this server as a source (or passing it to the gem command), any Java library in the world is installable as a gem. Let me repeat that: <span class="caps">ANY</span> Java library in the world, installable as a gem. This means you can also use Maven artifacts as dependencies in regular Ruby gems, and it additionally means we won't have to re-release jar files into their own duplicate gems on the standard repositories. It's very exciting, and we hope to have it ready for JRuby 1.5."</em> You can get it here: <a href="http://github.com/jruby/maven_gem" rel="nofollow">maven_gem at github</a><a></a><br><small><em>by <a href="http://www.datazoo.de" rel="nofollow">datazoo</a></em></small></p>
<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3563">light_mongo: a lightweight MongoDB object persistence layer for Ruby</a></strong> <a href="http://github.com/elliotcm/light_mongo">LightMongo</a> is a lightweight <a href="http://www.mongodb.org/display/DOCS">Mongo</a> object persistence layer for Ruby which makes use of Mongo's features rather than trying to emulate ActiveRecord. It's an interesting alternative to the already popular and somewhat awesome <a href="http://github.com/jnunemaker/mongomapper">MongoMapper.</a><br><small><em>by <a href="http://peterc.org/">PeterCooper</a></em></small></p>
<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3562">Multiple CPUs =&gt; Parallel Testing (now for every test-suite)</a></strong> <a href="http://github.com/grosser/parallel_tests">Parallel tests</a> now suppots any kind of test-suite(Test/RSpec/Cucumber), speeds up e.g. action_pack test-suite by 200%. Finally a good excuse for buying Dual/Quad-Core. <a href="http://github.com/grosser/parallel_tests">Speedup those tests, go parallel!</a><br><small><em>by <a href="http://pragmatig.wordpres.com">grosser</a></em></small></p>
<p class="rflink"><strong><a class="rf" href="http://rubyflow.com/items/3537">Schema-Free MySQL vs NoSQL (with help from Ruby)</a></strong> There is no reason why we can't have a schema-free MySQL engine to compete with the NoSQL solutions. A look at what "schema-free" and "document-oriented" actually means, and the <a href="http://www.igvita.com/2010/03/01/schema-free-mysql-vs-nosql/">ruby code to make it work</a>.<br><small><em>by <a href="http://www.igvita.com">igrigorik</a></em></small></p>
