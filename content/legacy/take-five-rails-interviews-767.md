<p>The <a href="http://blog.fiveruns.com/takefive">Take Five series on the FiveRuns blog</a> is a set of generally interesting interviews with people involved in the Ruby on Rails community: <a href="http://blog.fiveruns.com/2008/1/12/takefive-five-questions-with-chad-fowler">Chad Fowler</a>, <a href="http://blog.fiveruns.com/2008/1/18/takefive-five-questions-with-michael-cote">Michael Coté</a>, <a href="http://blog.fiveruns.com/2008/1/25/rails-application-monitoring-takefive-five-questions-with-peter-cooper">Peter Cooper</a>, <a href="http://blog.fiveruns.com/2008/2/1/rails-application-monitoring-takefive-five-questions-with-jeffrey-krause">Jeffrey Krause</a>, <a href="http://blog.fiveruns.com/2008/2/8/rails-application-monitoring-takefive-five-questions-with-robert-dempsey">Robert Dempsey</a>, <a href="http://blog.fiveruns.com/2008/2/15/rails-application-monitoring-takefive-five-questions-with-todd-barr">Todd Barr</a>, and <a href="http://blog.fiveruns.com/2008/2/22/rails-application-monitoring-takefive-five-questions-with-pat-eyler">Pat Eyler</a>. Each is asked five questions (selected from a set of about fifteen) with many answering the same questions with radically differing answers. There's a focus on questions involving Rails' position in the "enterprise," so if that topic interests you, take a closer look.</p>
<p>The series is on-going, and I believe FiveRuns are keeping their eyes peeled for more people to interview, so if you're interested, get in touch with them or leave a comment here and I'll let them know.</p>
