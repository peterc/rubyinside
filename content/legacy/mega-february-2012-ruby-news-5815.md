I forgot to post the weekly Ruby news updates from <a href="http://rubyweekly.com/">Ruby Weekly</a> to Ruby Inside in February so.. here's a mega roundup of all that was new in the Ruby and Rails worlds in February 2012. I'll try to keep posting every week from here on - sorry.</p>
<p>Highlights include: <strong>a new Ruby 1.9.3 release, REE's end of life, Spree 1.0, some Rails 4 news, Devise 2.0, a new private gem hosting service</strong>.. and that's just scratching the surface :-) Enjoy! (And don't forget to <a href="http://rubyweekly.com/">subscribe to Ruby Weekly</a> if you want to receive something like this every week via e-mail or <a href="http://rubyshow.com/">The Ruby Show</a> if you want it in podcast/audio form.)</p>
<h3>Headlines</h3>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://preview.ruby-lang.org/en/news/2012/02/16/ruby-1-9-3-p125-is-released/" style="font-weight: bold;">Ruby 1.9.3-p125 Released</a><br>
Patchlevel 125 of Ruby 1.9.3 is the latest production release of MRI. It adds LLVM/clang support (ideal for OS X Lion users), GCC 4.7 support, and includes security fixes in the OpenSSL extension.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://blog.phusion.nl/2012/02/21/ruby-enterprise-edition-1-8-7-2012-02-released-end-of-life-imminent/" style="font-weight: bold;">Ruby Enterprise Edition 1.8.7-2012.02 released; End of Life Imminent</a><br>
Phusion has unveiled the latest release of REE which is based on Ruby 1.8.7-p358 and RubyGems 1.8.15 and is compatible with XCode 4 and OS X Lion. However, REE is being slowly retired and no Ruby 1.9 version is forthcoming for several reasons.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://jruby.org/2012/02/22/jruby-1-6-7" style="">JRuby 1.6.7 Released: More 1.9 Fixes</a></p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://news.railstutorial.org/a-full-draft-of-the-ruby-on-rails-tutorial-2n" style="font-weight: bold;">Full Draft of 'Rails Tutorial' 2nd Edition is Now Available</a><br>
Michael Hartl, author of 'Rails Tutorial', has finished a draft of a new Rails 3.2-focused edition of the popular book and Web site. This is a popular way to learn Rails and it's only getting better!
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://spreecommerce.com/blog/2012/02/09/spree-1-0-0-released/" style="font-weight: bold;">Spree 1.0 Released (Rails-based E-commerce System)</a><br>
Spree is almost certainly the most popular, fully featured Rails-based e-commerce system and its creators are proud to announce the release of version 1.0.0. This follows the $1.5M seed funding of Spree's parent company in October 2011.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://blog.jetbrains.com/ruby/2012/02/rubymine-4-is-here-to-make-you-feel-the-productivity/" style="font-weight: bold;">RubyMine 4.0 (Ruby IDE) Released</a><br>
RubyMine is a popular Ruby and Rails IDE by JetBrains (the folks behind IntelliJ IDEA). A focus has been put on improving its performance and UI, but it now also supports all of Rails 3.2 features, including CoffeeScript compilation right from the IDE.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://refinerycms.com/blog/refinery-cms-2-0-0-released" style="font-weight: bold;">Refinery CMS 2.0.0 Released</a><br>
Refinery CMS is a Rails based CMS which in its new 2.0.0 incarnation is now fully Rails 3.2 and asset pipeline compliant. This post sums up some of the changes. Congrats to them.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://weblog.rubyonrails.org/2012/2/26/edge-rails-patch-is-the-new-primary-http-method-for-updates" style="font-weight: bold;">For Rails 4, PATCH Is The New Primary HTTP Method for Updates</a><br>
Xavier Noria of the Rails core team shares a new development that's in edge Rails (and due to be released in Rails 4): switching to using HTTP's 'PATCH' verb for making partial updates to resources.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://www.codeschool.com/courses/rails-testing-for-zombies" style="">Code School Unleashes 'Rails Testing for Zombies'</a></p>
</div>
<div style="margin: 16px 0px;" class="item">
  <a href="http://radar.oreilly.com/2012/01/with-govuk-british-government.html"><img src="http://s3.amazonaws.com/nlga/uploads/item/image/1882/thumb_ukruby.png" width="133" height="100" style="float: right; margin-left: 14px; margin-bottom: 16px; border: 1px solid #1173c7"></a>
<p><a href="http://radar.oreilly.com/2012/01/with-govuk-british-government.html" style="font-weight: bold;">British Government's New Portal Built on Ruby, Rails and Sinatra</a><br>
The British Government has launched the beta of 'gov.uk', a new attempt to centralize government Web sites. Ruby is not a focus of this article but it's revealed that Ruby makes up 'most of the application code' with a mixture of Rails and Sinatra.
</p>
</div>

<h3>Code</h3>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://blog.plataformatec.com.br/2012/01/devise-2-0-released/" style="font-weight: bold;">Devise 2.0 Released: The Flexible Auth System Grows Up</a><br>
Devise 2.0 is here but it's aimed at deprecations rather than new features. Nonetheless, it adds Rails 3.2 support and supports e-mail reconfirmation. Beware though, it now only supports Rails 3.1 and above.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://mperham.github.com/sidekiq/" style="font-weight: bold;">Sidekiq: Efficient Resque-compatible Message Processing for Rails 3</a><br>
Got lots of Resque processes running at once? Sidekiq offers 'simple, efficient message processing' for Rails 3 applications and due to its actor-based concurrency, a single Sidekiq process could do the work of many Resque ones.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="https://github.com/solnic/virtus" style="font-weight: bold;">Virtus 0.2.0: Attributes on Steroids for Plain Old Ruby Objects</a><br>
Virtus is a partial extraction of the DataMapper Property API with which you can extend your Ruby objects with attributes that require data type coercions. Handier than it sounds; see the README examples.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://rubydoc.info/gems/alterego/0.0.3/frames" style="font-weight: bold;">AlterEgo: Hosted Two Factor Authentication for your Web App</a><br>
A gem to access AlterEgo, a two factor authentication service built by MailChimp, the folks I use to send Ruby Weekly each week :-)
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="https://github.com/ExtractMethod/prickle" style="font-weight: bold;">Prickle: A Simple DSL Extending Capybara</a><br>
Capybara has become the de facto Ruby acceptance test framework for web apps and Prickle takes things a step further with extra methods for finding elements, performing actions, and more.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="https://github.com/mendicant-university/newman" style="font-weight: bold;">Newman: A Microframework for Email-based Apps</a><br>
Still in a highly experimental state, Newman is a micro-framework aiming to do for email-based applications what Rack and Sinatra have done for web apps. Gregory Brown leads the project.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
  <a href="https://github.com/maccman/headsup"><img src="http://s3.amazonaws.com/nlga/uploads/item/image/1875/thumb_hud.png" width="133" height="100" style="float: right; margin-left: 14px; margin-bottom: 16px; border: 1px solid #1173c7"></a>
<p><a href="https://github.com/maccman/headsup" style="font-weight: bold;">Heads Up: A Rails and JS Powered Desktop Heads Up Display</a><br>
Heads Up is a simple HUD showing your calendar items for today, your unread emails and custom notes. The interesting part is it's an OS X app, yet it's built on Rails, JavaScript (using Spine), Rack::Offline and MacGap.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="https://gist.github.com/1688857" style="font-weight: bold;">A Patched Ruby 1.9.3-p0 for A 30% Faster Rails Boot</a><br>
A somewhat adhoc set of patches to boost the performance of Ruby 1.9.3-p0 have been floating around on GitHub this week. Tread with caution but it may appeal to you. Hopefully the ideas will make their way into MRI proper.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="https://github.com/banister/plymouth" style="font-weight: bold;">Plymouth: Start Pry in the Context of a Failed Test</a><br>
Pry is an awesome alternative to the IRB Ruby console and Plymouth will automatically bring up a Pry session when a test fails in your suite, putting you right into the context of the failure. Supports Bacon, Minitest and RSpec.
</p>
</div>
<div style="margin: 16px 0px;" class="item">
<p><a href="http://blog.plataformatec.com.br/2012/02/simpleform-2-0-bootstrap-for-you-with-love/" style="font-weight: bold;">SimpleForm 2.0: Rails Form Creator, Now With Twitter Bootstrap Support</a><br>
SimpleForm is a popular DSL for creating forms in Rails apps and version 2 is now out. It includes a new wrapper API to create custom input stacks and also includes some integration features for Twitter Bootstrap.
</p>
</div>
<div style="margin: 16px 0px;" class="item">

<p><a href="https://github.com/seamusabshere/unix_utils" style="font-weight: bold;">UnixUtils: Like FileUtils But For Using Unix Tools</a><br>
Like FileUtils, but provides access to zip, unzip, bzip2, bunzip2, tar, untar, sed, du, md5sum, shasum, cut, head, tail, wc, curl, etc. The README explains why you wouldn't want to just spawn these yourself.
</p>
