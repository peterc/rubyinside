<p>After two successful <a href="preview-release-of-rails-tm-20-goes-live-615.html">release candidate</a> releases, <a href="http://rubyonrails.org/">Ruby on Rails</a>™, a popular Ruby-based Web application development framework in a similar vein to <a href="http://www.merbivore.com/">Merb</a> or <a href="http://www.nitroproject.org/">Nitro</a>, has successfully <a href="http://weblog.rubyonrails.org/2007/12/7/rails-2-0-it-s-done">made it to the final release of version 2.0</a>. Previously called an "evolutionary rather than revolutionary" step, Rails™ 2.0 nonetheless packs a whole ton of new features that make it a worthwhile upgrade from 1.x. Unsurprisingly, David Heinemeier Hansson, creator of Rails, has the best writeup in the <a href="http://weblog.rubyonrails.org/2007/12/7/rails-2-0-it-s-done">official launch announcement</a>.</p>
<p>Also bound to be of interest to Rails 2.0 users:</p>
<p><a href="http://www.slashdotdash.net/articles/2007/12/03/rails-2-upgrade-notes">Rails 2 Upgrade Notes</a> - Some notes from Ben Smith covering the process of upgrading an application from using Ruby 1.2.6 to Rails 2.0. It includes a particularly useful Rake task to check your code for certain deprecated features.</p>
<p><a href="http://www.slashdotdash.net/articles/2007/12/04/iphone-on-rails-creating-an-iphone-optimised-version-of-your-rails-site-using-iui-and-rails-2">iPhone on Rails</a> - Ben Smith continues with a great demonstration of how to easily build iPhone-targeted applications using Rails 2.0.</p>
<p><a href="http://ryandaigle.com/articles/2007/12/7/rails-2-0-final-released-summary-of-features">Summary of Major Rails 2 Features</a> - Ryan Daigle wraps up a stunning collection of "Edge Rails" blog posts demonstrating Rails 2.0 features many months in advance with a mega link post.</p>
