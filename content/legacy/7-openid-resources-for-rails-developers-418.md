<p>Decentralized identity system, <a href="http://openid.net/">OpenID</a>, appears to be the latest flavor of kool-aid permeating the Rails universe. OpenID makes it possible for users to identify themselves by using URLs that relate to a site where they have already logged in. Your application can then check this with the remote site and then use that status to tie that confirmed identity to their identity within your system. As much for my own reference as anything else, here are the main bits and pieces floating around regarding Rails and OpenID lately:</p>
<p>1) David Heinemeier Hansson started off by <a href="http://www.loudthinking.com/arc/000606.html">talking about how 37signals are looking at using OpenID</a>.</p>
<p>2) Dan Webb wrote an excellent tutorial, "<a href="http://www.danwebb.net/2007/2/27/the-no-shit-guide-to-supporting-openid-in-your-applications">The No Shit Guide To Supporting OpenID In Your Applications</a>", showing how to, well, use OpenID within Rails applications.</p>
<p>3) Ben Curtis demonstrates <a href="http://www.bencurtis.com/archives/2007/03/rails-openid-and-acts-as-authenticated/">how to tie in OpenID with the popular Acts As Authenticated plugin</a>.</p>
<p>4) The <a href="http://agilewebdevelopment.com/plugins/openidauthentication">OpenID Authentication plugin</a> wraps around the Ruby-OpenID gem and makes OpenID integration from Rails simple. Code examples are provided.</p>
<p>5) John Nunemaker <a href="http://railstips.org/2007/2/28/openid-and-rails">points</a> us to the <a href="http://wiki.rubyonrails.org/rails/pages/OpenID">Ruby on Rails wiki page for OpenID</a> that provides links to several other resources.</p>
<p>6) Way back in the summer of 2006, EastMedia and Verisign <a href="http://identity.eastmedia.com/identity/show/Rails+OpenID+Server">developed a Rails-based open source OpenID </a><em><a href="http://identity.eastmedia.com/identity/show/Rails+OpenID+Server">server</a></em> that was released under the Apache Heraldry project. Verisign use it to power <a href="http://pip.verisignlabs.com/">their PIP system</a>.</p>
<p>7) Dag Rorek Arneson <a href="http://%5C">built a Rails OpenID guestbook</a> that's still available (but very slow) at <a href="http://rorek.org:3001/">http://rorek.org:3001/</a>.</p>
<p><strong>NEW! </strong>8) Bernie Thompson has put together <a href="http://leancode.com/openid-for-rails/">a kick ass page of Rails and OpenID resources</a> as well as a <a href="http://video.google.com/videoplay?docid=-6681175422710559031">screencast</a> that quickly demonstrates the power of OpenID.</p>
