<p><img src="/assets/2008/06/railsconfgfx.jpg" width="350" height="132" alt="railsconfgfx.jpg" style="border:1px #000000 solid;"></p>
<p><a href="http://en.oreilly.com/rails2008/public/content/home">RailsConf 2008</a> - the primary conference for Rails developers - took place over the last few days (May 29th to June 1st, 2008). By all accounts, everyone had a great time, but not everyone could attend so here's a casual roundup of what happened.</p>
<p>The best overall "walkthrough" of the conference I've seen has been by Drew Blas who put together a great set of blog posts covering: <a href="http://drewblas.com/2008/05/30/railsconf-2008-friday-summary/">Friday morning</a>, <a href="http://drewblas.com/2008/05/31/railsconf-2008-friday-afternoon-summary/">Friday afternoon</a>, <a href="http://drewblas.com/2008/05/31/railsconf-2008-friday-evening-summary/">Friday evening</a>, <a href="http://drewblas.com/2008/05/31/railsconf-2008-saturday-morning-summary/">Saturday morning</a>, <a href="http://drewblas.com/2008/05/31/railsconf-2008-saturday-afternoon-summary/">Saturday afternoon</a>, <a href="http://drewblas.com/2008/05/31/railsconf-2008-saturday-evening-summary/">Saturday evening</a>, <a href="http://drewblas.com/2008/06/01/railsconf-2008-sunday-morning-summary/">Sunday morning</a> and <a href="http://drewblas.com/2008/06/01/railsconf-2008-sunday-afternoon-summary/">Sunday afternoon</a>. He even put together <a href="http://drewblas.com/2008/06/01/railsconf-2008-final-summary/">a final summary</a>. Great work Drew! For Italian readers, <a href="http://factory.wavegroup.it/blog/railsconf-2008/">RailsConfLive</a> is a remarkable resource too. Alternatively, Gregg Pollack's <a href="http://railsenvy.com/2008/6/2/Railsconf-videos">set of videos, including "Railsconf in 36 minutes."</a></p>
<p><big><strong>Interviews</strong></big></p>
<p><img src="/assets/2008/06/dhhinterview.jpg" width="258" height="194" alt="dhhinterview.png"></p>
<p>Quite a few informal interviews took place at the conference, mostly by the guys from <a href="http://www.gotthingsdone.com/">GotThingsDone</a>. Worth watching are <a href="http://youtube.com/watch?v=4yXzptoJD8k&amp;feature=related">Geoffrey Grosenbach</a>, <a href="http://youtube.com/watch?v=Au6Q8c1i2GI&amp;feature=related">John Straw</a>, <a href="http://youtube.com/watch?v=TykHa0ZePIk&amp;feature=related">Ola Bini</a>, <a href="http://youtube.com/watch?v=OR0prVLD3rk&amp;feature=related">Ryan Singer</a>, <a href="http://youtube.com/watch?v=q1jAKk5Yax0&amp;feature=related">Joel Spolsky</a>, <a href="http://youtube.com/watch?v=wCEKSXZUM_w&amp;feature=related">David Heinemeier Hansson</a>, <a href="http://youtube.com/watch?v=uRvSo7Vj-0o&amp;feature=related">Jeremy Kemper</a>, and <a href="http://youtube.com/watch?v=-Reg0yhJw4s&amp;feature=related">Chris Wanstrath</a>.</p>
<p>Gregg Pollack of RailsEnvy also interviewed <a href="http://www.vimeo.com/1104211">Obie Fernandez</a>, <a href="http://www.vimeo.com/1104389">Chu Yeow</a>, <a href="http://www.vimeo.com/1104149">Amy Hoy</a>, <a href="http://www.vimeo.com/1104408">Michael "Koz" Koziarski</a>, <a href="http://www.vimeo.com/1104432">David Heinemeier Hansson</a>, <a href="http://www.vimeo.com/1104465">Bruce Williams</a>, and <a href="http://www.vimeo.com/1104493">Ninh Bui</a>.</p>
<p><big><strong>Selection of Presentations</strong></big></p>
<p><strong><span style="font-weight: normal;"><span style="font-style: italic;">Note: You can</span> <a href="http://en.oreilly.com/rails2008/public/schedule/proceedings"><span style="font-style: italic;">download the presentation files used</span></a> <span style="font-style: italic;">by some of the speakers from the official RailsConf site.</span></span><br></strong></p>
<p><strong><span style="font-style: italic;">DHH Keynote:</span></strong> Thanks to Daniel Wanja, you can <a href="http://onrails.org/articles/2008/05/31/railsconf-2008-david-heinemeier-hanssons-keynote-video">watch David Heinemeier Hansson's keynote presentation</a>. It's a bit of a shaky video and only made up of extracts, so it jumps a bit too. I imagine a better video will be available soon. Essentially, DHH moves from technical concerns to becoming a self-help and health advisor.</p>
<p><strong><span style="font-style: italic;">RailsConf 2008 Welcome Keynote:</span></strong> The welcome keynote, featuring Chad Fowler, David Heinemeier Hansson and Joel Spolsky is <a href="http://youtube.com/watch?v=ZrOOuWjhOFQ">up on YouTube</a>.</p>
<p><strong><span style="font-style: italic;">Ruby Heroes:</span></strong> RailsConf 2008 marked the inauguration of the "<a href="ruby-hero-awards-nominate-your-ruby-hero-864.html">Ruby Hero</a>" award. The ultimate winners were Evan Weaver, Tom Copeland, James Edward Gray II, Ilya Grigorik, Yehuda Katz and Ryan Bates. You can <a href="http://www.vimeo.com/1099180?pg=embed&amp;sec=1099180">watch the video of the awards presentation</a> at Vimeo.</p>
<p><strong><span style="font-style: italic;">Getting Git:</span></strong> Scott Chacon did <a href="http://jointheconversation.org/2008/06/02/railsconf-git-talk/">a talk on "Getting Git".</a> It perhaps sets the record for number of slides at 520 covered in 55 minutes.</p>
<p><strong><span style="font-style: italic;">The Other mod_rails - Deployment with JRuby:</span></strong> Nick Sieger did <a href="http://blog.nicksieger.com/articles/2008/05/31/railsconf-slides">a presentation</a> about using JRuby to deploy Rails applications.</p>
<p><strong><span style="font-style: italic;">Pastie:</span></strong> Josh Goebel did <a href="http://blog.pastie.org/2008/05/pastie-lightnin.html">a presentation</a> about the popular <a href="http://pastie.org">Pastie</a> source-code-pasting site.</p>
<p><strong><span style="font-style: italic;">Microapps for Fun and Profit:</span></strong> Erik Kastner talked about <a href="http://metaatem.net/2008/05/30/my-railsconf-talk">developing very small "microapps"</a> using Rails.</p>
<p><strong><span style="font-style: italic;">Assembling Pages Last:</span></strong> Aaron Batalion <a href="http://blog.hungrymachine.com/2008/5/31/railsconf-2008-presentation-assembling-pages-last">presented</a> on edge caching with Rails.</p>
<p><strong><span style="font-style: italic;">Advanced RESTful Rails:</span></strong> Ben Scofield <a href="http://www.slideshare.net/viget/advanced-restful-rails-439639">covers the use of REST with Rails</a> and includes lots of simple code examples. Perhaps the slides lack a lot of what happened in the presentation, but this seems like basic REST stuff than "advanced" so it's well worth a read even if you don't use REST yet.</p>
<p><strong><em>Scaling Ruby from the Inside Out:</em></strong> Ezra Zygmuntowicz <a href="http://brainspl.at/articles/2008/06/02/introducing-vertebra">on scaling Ruby, and Vertebra</a> a new "next generation cloud computing / automation framework."</p>
<p><strong><span style="font-style: italic;">Scaling Rails Panel:</span></strong> A panel of Blaine Cook, Bradley Taylor, Ezra Zygmuntowicz, Jim Meyer and Kevin Lawver <a href="http://www.vimeo.com/1101097">talk about the issues</a> when scaling Rails applications.</p>
<p><strong><span style="font-style: italic;">Railsconf in 36 Minutes:</span></strong> Not a presentation, but roving reporter <a href="http://railsenvy.com/2008/6/2/Railsconf-videos">Gregg Pollack went around and asked a lot of the presenters to summarize their talks</a> in 30 seconds. It makes for a great summary of the conference. It features the interviews linked above too.</p>
<p><strong><em>Lightning Talk Summaries:</em></strong> A <a href="http://www.urbanhonking.com/ideasfordozens/2008/06/railsconf-2008-lightning-talks.html">summary of the lightning talks</a> given at the conference.</p>
<p><big><strong>The RailsConf Message Board</strong></big></p>
<p><a href="http://flickr.com/photos/viniciusteles/2538507982/"><img src="/assets/2008/06/railsconfboard08.jpg" width="350" height="262" alt="railsconfboard08.jpg" style="border:1px #000000 solid;"></a></p>
<p>It seems to be a tradition at Ruby / Rails conferences to put up a big "message board" for people to write / graffiti on as they see fit. RailsConf 08 was no exception. Vinícius Manhães Teles has taken a great high-resolution <a href="http://flickr.com/photos/viniciusteles/2538507982/">picture of the board</a>. Messages include promotion for a new Gem called <a href="http://github.com/chad/gitjour/tree/master">gitjour</a>, <a href="http://codesmack.com/">codesmack.com</a> t-shirts and a crazy number of requests for Rails developers (Zappos.com, Engine Yard, AideRSS, Synaptic Mash, Grockit, XMinute and Intridea are all hiring!).</p>
<p><big><strong>Announcements</strong></big></p>
<p><strong><span style="font-style: italic;">Rails 2.1 Launched:</span></strong> <a href="http://weblog.rubyonrails.com/2008/6/1/rails-2-1-time-zones-dirty-caching-gem-dependencies-caching-etc">Rails 2.1 was officially launched</a>. More about this on Ruby Inside in the next couple of days!</p>
<p><strong><span style="font-style: italic;">Passenger 2.0 - Now supporting Rack:</span></strong> Phusion <a href="http://www.railsjedi.com/posts/52-The-Holy-Grail-for-Rails-Deployment">announced</a> that Passenger 2.0 (now supporting Rack!) and "Ruby Enterprise" 1.0 would be released today.</p>
<p><strong><span style="font-style: italic;">MagLev - A New Ruby VM:</span></strong> Avi Bryant and Bob Walker of GemStone <a href="http://www.infoq.com/news/2008/05/MagLevAtRailsConf">revealed</a> details surrounding their cutting-edge Ruby virtual machine, MagLev. Drew Blas posts with <a href="http://drewblas.com/2008/05/31/railsconf-2008-surprise-of-the-day-maglev/">his opinions direct from the talk</a>. Avi Bryant has <a href="http://www.avibryant.com/2008/06/maglev-recap.html">posted some more detailed notes</a>. Giles Bowkett <a href="http://gilesbowkett.blogspot.com/2008/06/maglev-jruby-and-rubinius-who-will-win.html">steps it all up a bit</a>.</p>
<p><strong><span style="font-style: italic;">TuneUp:</span></strong> FiveRuns <a href="http://blog.fiveruns.com/2008/5/30/announcing-fiveruns-tuneup-community-supported-application-performance-profiling">announced TuneUp</a>, a community-supported application performance profiling application.</p>
