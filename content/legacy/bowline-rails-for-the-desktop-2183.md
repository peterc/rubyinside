<p><img style="float:left; margin-right:12px; margin-bottom:12px;" src="/assets/2009/08/bowline-twitter-c.png" alt="bowline"><a href="http://github.com/maccman/bowline">Bowline</a> is a new <a href="http://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller">MVC</a> GUI framework for developing cross-platform desktop applications using Ruby, HTML, CSS and Javascript. The author, <a href="http://eribium.org/">Alex MacCaw</a>, aims to make building a desktop app as simple as creating a <a href="http://rubyonrails.org/">Rails</a> site.</p>
<p>Unlike pure Ruby desktop libraries like <a href="http://shoooes.net/">Shoes</a>, Bowline builds on <a href="http://www.appcelerator.com/products/titanium-desktop/">Appcelerator Titanium</a>, an SDK for building desktop apps that basically gives you a Webkit window and a set of APIs for interacting with various operating systems. This lets you develop for the desktop using familiar web technologies, including CSS3 and HTML5 (think Adobe Air!).</p>
<p>To allow your Ruby code to interact with your HTML, Bowline introduces the concept of "binders" which can update your views as the underlying data changes (a bit like controllers in Rails).  Bowline also includes other ideas analogous to parts of the Rails framework, such as generators, helpers and models.</p>
<p>In <a href="http://leadthinking.com/191-bowline-a-ruby-gui-framework">this blog post</a> Alex MacCaw shows off a simple Twitter client he built using Bowline - you can download it and continue to play on your own if you like. Nice work!</p>
