<h3>Jeremy McAnally - One of Ruby’s Busiest Hackers of 2008</h3>
<p><img src="/assets/2008/12/jeremym.png" width="109" height="146" alt="jeremym.png" style="float:right; margin-top:12px; margin-bottom:12px; margin-left:12px;">Jeremy McAnally (Twitter <a href="http://twitter.com/jeremymcanally/">@jeremymcanally</a>) is a Huntsville, Alabama based Ruby developer who has been developing software for about 10 years and now works for popular consultancy <a href="http://entp.com/">ENTP</a>. He takes <em>Top Hitter</em> (no, not <em>Hitler</em>!) for being, in Ruby Inside's opinion, one of the most consistent releasers, challengers, and driving forces in the Ruby and Rails worlds this year. His releases span many media - two books (<a href="http://www.manning.com/mcanally/">Ruby In Practice</a> and <a href="http://www.humblelittlerubybook.com/">The Humble Little Ruby Book</a>), a magazine (<a href="http://www.therubyist.com/">The Rubyist</a>), libraries (<a href="http://github.com/jeremymcanally/context/tree/master">context</a>, <a href="http://github.com/jeremymcanally/matchy/tree/master">matchy</a>, <a href="http://dcov.rubyforge.org/">dcov</a>, and more), and a blog (his blog - <a href="http://www.omgbloglol.com/">omg blog!! lol!!</a>).</p>
<p>Jeremy cites his overriding goal as making things simpler by solving complicated problems with beautiful implementations. In this sense he has done very well. His latest significant project, <a href="http://github.com/jeremymcanally/rg">rg</a>, a Rails application generation template system, has been <a href="http://github.com/rails/rails/commit/e8cc4b116c460c524961a07da92da3f323854c15">integrated into the Rails core</a> and will make life a lot easier for Rails developers in the future. Jeremy was also the driver behind one of only two successful efforts to launch a Ruby print magazine in 2008 with <a href="http://www.therubyist.com/">The Rubyist</a> (the other being <a href="http://it-republik.de/railsway/magazin-ausgaben/Ruby-on-Rails-000281.html">the German Rails Way</a>). With <a href="http://github.com/jeremymcanally/context/tree/master">context</a> and <a href="http://github.com/jeremymcanally/matchy/tree/master">matchy</a>, Jeremy has also taken a good shot at making Test::Unit an attractive alternative to RSpec.</p>
<p>If we all kept releasing and trying as many things as Jeremy has in the last couple of years, the Ruby community would be awash with all sorts of amazing projects. For this, he easily takes Ruby Inside's slightly awkwardly named <em>Top Hitter</em> accolade.</p>
<h3>Mini Interview</h3>
<p>We caught up with Jeremy for a short interview to find out what he's been up to, what he has planned, and who he finds interesting in the Ruby world..</p>
<p><strong><em>Ruby Inside:</em> How and when did you get into Ruby?</strong></p>
<blockquote>
<p><em>Jeremy McAnally:</em> I was working on a project and needed dynamic typing. I found Ruby and *loved* it, but the serial library wasn’t working with my hardware, so I ended up using Python instead. I rediscovered Ruby a few months later when I saw an article about Rails and haven’t looked back since (except one unfortunate liaison with PHP but we won’t talk about that!).</p>
</blockquote>
<p><strong>How have your Ruby &amp; Rails related projects progressed in 2008? Any surprises?</strong></p>
<blockquote>
<p>I’ve had a lot of surprises this year. I honestly didn’t think <a href="http://www.therubyist.com/">The Rubyist</a> would ever actually come to fruition, but here we are. One issue out and another on the way.</p>
<p>After a lot of false starts for projects (an RDoc replacement named docr, a Rake replacement named lake, and a few others), I ended up with a few solid releases of some testing tools (<a href="http://github.com/jeremymcanally/context/tree/master/tree/master">context</a>, <a href="http://github.com/jeremymcanally/matchy/tree/master">matchy</a>, and <a href="http://github.com/jeremymcanally/stump/tree/master">stump</a>). They’re still taking shape, but a lot of people seem to really enjoy using them as much as I do. I’m really interested to see where they go; it makes me feel good to be using simple code to write beautiful tests!</p>
</blockquote>
<p><strong>What plans / outlook / projects do you have for 2009?</strong></p>
<blockquote>
<p>I want to keep pushing my testing tools forward, but I also have a few other places I want to go. I want to get back to contributing to Rails (and I’m working on a few interesting things in that realm right now); I want to work on simplifying many of the core tools we depend on these days (and work alongside those who have already started doing so, such as the folks who’ve release minitest, thor, and so on). Other than that, I’ll have to see what presents itself. :)</p>
<p>Outside of Ruby, I plan on doing some iPhone work also. I’ve started hacking a few apps, but I want to extract some pieces of those into some open source libraries. There’s a very small amount of OSS code out there for iPhone, which means there’s plenty of territory to cover for those who are willing to do the work.</p>
</blockquote>
<p><strong>What do you think other Rubyists should be learning or checking out in the new year?</strong></p>
<blockquote>
<p>I’m not totally sure. I cast a fairly broad eye across the Ruby ecosystem, but I think the most interesting thing I see right now is <a href="http://www.adhearsion.com/">Adhearsion</a>. It’s sort of a sleeper project, but I think when it hits critical mass, it’s going to be huge.</p>
</blockquote>
<p><strong>Have any Rubyists have made an impact on your work in 2008?</strong></p>
<blockquote>
<p>I’ve been really interested to see all the fun and interesting Ruby tricks out there (simple example: the new #try stuff in Rails). Some of them are really elegant, and they’ve taught me a lot about Ruby objects and how to effectively expand their behavior. As for blogs, I’ve really been loving <a href="http://afreshcup.com/">Mike Gunderloy’s blog</a>. He’s a fairly recent addition to my blogroll, but he always has interesting links. <a href="http://blog.bleything.net/">Ben Bleything</a> also has a lot of interesting projects, and, as I’m sure you’re ware, <a href="http://gilesbowkett.blogspot.com/">Giles</a> always has really interesting and sometimes disturbing posts.</p>
</blockquote>
<p>Thanks Jeremy!</p>
