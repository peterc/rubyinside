<p>On the back of <a href="http://glu.ttono.us/articles/2006/08/30/guide-things-you-shouldnt-be-doing-in-rails">Kevin Clark's "Things You Shouldn't Be Doing in Rails"</a>, Geoffrey Grosenbach has created a plugin called '<a href="http://nubyonrails.com/articles/2006/08/31/deprecated-plugin-find-old-rails-code">deprecated</a>' that, with a single rake task, will <a href="http://nubyonrails.com/articles/2006/08/31/deprecated-plugin-find-old-rails-code">alert you to all of the old and deprecated code in your application</a>. This is perfect for bringing those old applications up to date and will help you catch all of the changes necessary, such as using flash and session instead of @flash and @session, etc.</p>
<p>I wonder how viable it would be to go even further and create something that could check for sloppy coding practices (rather than outdated practices) and fix spacing issues, etc.</p>
