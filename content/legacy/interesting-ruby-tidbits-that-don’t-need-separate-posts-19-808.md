<p><span style="font-size: 14px; font-weight: bold;">REST Client - Simple DSL / Ruby Library for Accessing REST Resources</span></p>
<p><img src="/assets/2008/03/rest-client.jpg" width="294" height="126" alt="rest-client.png"></p>
<p>Adam Wiggins of <a href="http://www.heroku.com/">Heroku</a> has made available <a href="http://rest-client.heroku.com/rdoc/">rest-client</a>, a new library that makes it ridiculously easy to interact with RESTful resources and APIs (over HTTP, naturally.)</p>
<p><big><strong>Racket - Raw Packet Manipulation with Ruby</strong></big></p>
<p><a href="http://spoofed.org/blog/archive/2008/03/racket_ruby_raw_packet_fun.html">Racket</a> is a new Ruby library that "crafts and analyzes" raw network packets. It's all pretty low level stuff, but a collection of straight forward <a href="http://spoofed.org/files/racket/examples/">examples</a> are provided to get you started.</p>
<p><big><strong>RubyDoes.Net - A Blog for IronRuby Fans</strong></big></p>
<p><a href="http://rubydoes.net/">RubyDoes.Net</a> is, effectively, like a Ruby Inside but for the .Net / Microsoft side of things. It focuses on IronRuby, Microsoft's .Net implementation of Ruby. IronRuby doesn't get a significant amount of press here on Ruby Inside, so for those with an interest in the Windows platform, RubyDoes.Net is a great blog to check out.</p>
<p><big><strong>RSpecced Haml Scaffolding for Rails</strong></big></p>
<p>Daniel Fischer writes in with news about the availability of <a href="http://blog.citrusbyte.com/2008/3/11/what-s-that-rspec-haml-scaffolds-yes-it-is">Haml scaffolding for Rails</a>. I'm not a Haml user myself, but it seems this adds a generator to Rails that makes it easy to generate RSpec-backed Haml-templated scaffolds for your Rails models. If so, and if you're a Haml fan, go get it!</p>
<p><big><strong>InLine - 100% Ruby Solution for Inline Editing at the Console</strong></big></p>
<p>Fabio Cevasco has put together <a href="http://www.h3rald.com/blog/inline-introduction">InLine</a>, a clever bit of Ruby code that makes it possible to create a console-based inline editing environment with pure Ruby. It seems a little bit like a hack, but it's a very clever bit of coding and bound to be of interest to those developing console-based applications.</p>
