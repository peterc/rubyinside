<h3>Is It JRuby? - Track which gems work on JRuby</h3>
<p><a href="http://isitjruby.com/"><strong>Is it JRuby?</strong></a> is a "fork" of Brightbox's <a href="http://github.com/brightbox/isitruby19/">Is It Ruby 1.9</a> that focuses on JRuby. As with the Ruby 1.9 site, Is It JRuby lets you track gems, allowing you to mark them as either working or not working with JRuby (yet) or to track reports from others. Remember that since these sites are community driven, the reports should never be taken as gospel and simply used as guidelines.</p>
<h3>Ruby Enterprise Edition's 3rd Sponsorship Campaign</h3>
<p>The Phusion guys (behind the awesome Passenger / mod_rails) are <a href="http://blog.phusion.nl/2009/05/27/ruby-enterprise-edition-third-sponsorship-campaign/">looking for contributions to their third sponsorship campaign for Ruby Enterprise Edition.</a> If you're not already aware, Ruby Enterprise Edition (aka REE) is a specially patched/adapted version of the MRI (Matz's Ruby Interpreter) that uses less memory, has a tweaked garbage collector and memory allocator, and several other performance enhancements. The target this time is $9000 and they have about $6000 left to go.</p>
<p>As an aside, Mike Perham's article, <a href="http://www.mikeperham.com/2009/05/25/memory-hungry-ruby-daemons/">Memory-hungry Ruby daemons</a>, demonstrates how REE can prove useful in reducing memory use on long-running daemons.</p>
<h3>MacRuby - Changing the Ruby Ecosystem?</h3>
<p>In <a href="http://merbist.com/2009/05/27/macruby-changing-the-ruby-ecosystem/">MacRuby, Changing the Ruby Ecosystem</a>, Matt Aimonetti writes about how <a href="http://www.macruby.org/">MacRuby</a> - the Apple sponsored, open source, Objective C driven Ruby implementation - fits into the Ruby ecosystem and, ultimately, how the big developments coming along could change it. Key points include MacRuby's <a href="http://rubyspec.org/">rubyspec</a> compliance, that MacRuby doesn't require OS X, and MacRuby's potentially explosive performance.</p>
<p><img src="/assets/2009/05/rupho.png" width="74" height="40" alt="rupho.png" style="float:right; margin-bottom:12px; margin-left:12px;"></p>
<h3>Beginner's iPhone Programming Workshop for Rubyists</h3>
<p>Dan Grigsby of leading iPhone development Web site <a href="http://www.mobileorchard.com/">Mobile Orchard</a> is putting on <a href="http://www.mobileorchard.com/inside">an iPhone programming workshop in Toronto</a> on July 9-10, <em>just</em> as <a href="http://futureruby.com/">FutureRuby</a> kicks off. If you're going to FutureRuby and fancy learning about iPhone development from a great teacher, this is a great chance. Even if you're <em>not</em> going to FutureRuby but are interested in the topic, check it out.</p>
<h3>Sketches - When Your Code Changes, It Gets Reloaded!</h3>
<p>It's easier to steal their already fine description, so.. <a href="http://sketches.rubyforge.org/">Sketches</a> allows you to create and edit Ruby code from the comfort of your editor, while having it safely reloaded in IRB whenever changes to the code are saved. As the name implies, this could prove pretty handy for basic "sketching" and prototyping of lightweight code.</p>
<h3>Engine Yard Takes Over Ruby 1.8.6 Maintenance</h3>
<p>With both Ruby 1.8.7 and Ruby 1.9.1, poor old Ruby 1.8.6 has been left in the dust support-wise. Despite this, it's still the most popular Ruby implementation out there and a lot of people depend on it. Given this, hosting company Engine Yard's team of engineers <a href="http://www.engineyard.com/blog/2009/engineyard_ruby186_maintenance/">have decided to take over the legacy maintenance duties</a>. Hats off to them!</p>
