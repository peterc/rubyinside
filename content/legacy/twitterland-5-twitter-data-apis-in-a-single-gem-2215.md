<p><img src="/assets/2009/08/twitter.png" width="64" height="64" alt="twitter.png" style="float:left; margin-right:12px; margin-bottom:12px;"><a href="http://locomotivation.squeejee.com/post/158183764/twitterland-all-the-best-twitter-related-apis-in-one-gem"><b>TwitterLand</b></a> (<a href="http://github.com/squeejee/twitterland/tree/master">Github repository</a>) is a single Ruby library that packages up access to, currently, 5 different Twitter-related data services. Note, however, it's <b>not</b> a library that allows you to post to Twitter - you'll want <a href="http://twitter4r.rubyforge.org/">Twitter4R</a> or John Nunemaker's <a href="http://twitter.rubyforge.org/">Twitter gem</a> for that.</p>
<p>More services are sure to be added to TwitterLand, but currently it makes it easy to get data from these five:</p>
<ul>
<li>
<a href="http://followcost.com/">Follow Cost</a> - A service that gives a "cost" on how annoying it will be to follow a specific Twitter user.</li>
<li>
<a href="http://twitter.grader.com/">Twitter Grader</a> - Provides info on how Twitter users measure up to each other in terms of popularity. It uses a user's number of followers, following, and updates to come up with a ranking.</li>
<li>
<a href="http://mrtweet.com/">Mr. Tweet</a> - A service that helps people find new people to follow on Twitter.</li>
<li>
<a href="http://twinfluence.com/">Twinfluence</a> - A "Twitter influence analyzer."</li>
<li>
<a href="http://twittercounter.com/">TwitterCounter</a> - The granddaddy of Twitter data mashups, TwitterCounter provides historical counts of users' following and follower counts (amongst other info).</li>
</ul>
<p>Notably, TwitterLand is developer Bradley Joyce's <i>first</i> gem, so congratulations to him.</p>
