<p><img src="/assets/2011/10/ruby2-0.gif" alt="" title="ruby2-0" width="93" height="111" class="alignnone size-full wp-image-5516" style="float: right; margin-left: 18px; margin-bottom: 18px">Today, Yusuke Endoh posted to the <a href="http://blade.nagaokaut.ac.jp/ruby/ruby-core/index.shtml">ruby-core</a> mailing list noting that matz has admitted him as a Ruby 2.0  "release manager" and as part of his work, he has come up with a tentative schedule for Ruby 2.0's release. You can <a href="http://blade.nagaokaut.ac.jp/cgi-bin/scat.rb/ruby/ruby-core/40301">read the full post here.</a></p>
<blockquote class="stylized"><p>I've been admitted as a 2.0 release manager by matz.  I'll do my<br>
best. As my first work, I'd like to announce a rough plan for 2.0<br>
release. Anyone who is interested in 2.0, especially who is thinking<br>
about feature proposal, should check it out.</p></blockquote>
<p><cite>Yusuke Endoh</cite></p>
<p>The short version of the timeline?</p>
<ul>
<li>
<strong>August 2012: 'Big feature' freeze.</strong> Any proposals and specs for 'major features' should be wrapped up for late August 2012 otherwise they will be deferred.</li>
<li>
<strong>October 2012: Feature freeze.</strong> A more conventional feature freeze.</li>
<li>
<strong>February 2013: 2.0 Release.</strong> The final version of 2.0 would, hopefully, be released.</li>
</ul>
<p>A release in February 2013 would have historical significance by being the 20th anniversary of Ruby's creation. Note that Yusuke corrected the date to February 24, 2013 in a later posting.</p>
<p>Another note, confirming what Yugui had already said, is that Ruby 2.0 is to be "100% compatible" with Ruby 1.9.3.</p>
<p><strong>If you want to learn more about Ruby 2.0, I recommend Ruby Inside's last post, <a href="ruby-2-0-implementation-work-begins-what-is-ruby-2-0-and-whats-new-5515.html">Ruby 2.0 Implementation Work Begins: What is Ruby 2.0 and What’s New?</a> where I summarized the anticipated new features in Ruby 2.0.</strong></p>
