<p>Almost a year ago I <a href="radiant-a-rails-cms-diamond-in-the-rough-52.html">posted</a> about <a href="http://radiantcms.org/">Radiant</a>, a then 'diamond in the rough' Rails content management system that was under development by John W. Long. Since then Radiant has come along in leaps and bounds and a significant release, namely that of version 0.6, <a href="http://radiantcms.org/blog/2007/04/24/radiant-0-6-lapidary-release/">has been made today</a>.<br>
<span style="font-size:0pt;"><br>
</span><img src="/assets/2007/04/radiant2.jpg" height="257" width="325" border="1" hspace="4" vspace="4" alt="Radiant2" title="radiant2.png"></p>
<p>Radiant is an open-source CMS developed with Ruby on Rails which features a new extension system enabling users to create <a href="http://dev.radiantcms.org/radiant/wiki/Extensions">third party add-ons</a> (which already include extensions to do backups, LDAP, virtual domains, vim editing, Flickr integration, page aggregation and more). It can be installed simply using RubyGems (gem install radiant) and getting an instance of the application up only takes a few lines at the command prompt. If you need a CMS and you're ready to roll with a solid Rails application, <a href="http://radiantcms.org/">Radiant</a> seems a pretty good place to start.</p>
